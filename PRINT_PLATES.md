# Print plates ‚ÄĒ drop-into-slicer layout

Generated by `make_print_plates.py` from `parts/*.stl` (which come from
`export_parts.py` ‚Üí `gripper.gen_step()`, CLOSED pose). Each part is rotated to
its **supportless** print orientation, seated on Z=0, then shelf-packed onto
build plates for a common bed.

> **REGENERATE AFTER ANY GEOMETRY CHANGE.** These plates are a *derived*
> artifact. When `gripper.py` geometry changes (e.g. the snap-fit / pin agent
> finalizes the barb), the integrator MUST re-run **both** steps:
> ```bash
> source /home/andre/.cad-venv/bin/activate
> python export_parts.py        # refresh parts/*.stl
> python make_print_plates.py   # refresh print_plates/
> ```
> Stale plates do not reflect the current design. `make_print_plates.py` wipes
> and rebuilds `print_plates/` on every run, so it is safe to re-run.

## What the script writes

- `print_plates/oriented/<part>.stl` ‚ÄĒ every unique part, **already rotated** to
  its print orientation with min-Z on the bed. Drop a single one in if you only
  need that part.
- `print_plates/plate_rigid_1.stl` ‚ÄĒ the 11 **PA12-GF** parts (body, cover, arms,
  followers, 4 axle dowels, input pinion shaft), packed.
- `print_plates/plate_petg_1.stl` ‚ÄĒ the 4 **PETG-HF** finger pins (`snap_pin_finger`),
  on their own plate: the split snap barb needs ductility, so these print in
  PETG-HF, not the brittle glass-filled nylon.
- `print_plates/plate_tpu_1.stl` ‚ÄĒ the two **TPU** Fin Ray fingers, packed
  **separately** because they print in a different material.

Plates are split by MATERIAL by design (PA12-GF / PETG-HF / TPU) ‚ÄĒ you swap
filament between the three plates.

## Bed & packing

- Default bed: **256 √ó 256 mm** (configurable: `--bed X Y`).
- Spacing between parts: **5 mm** (`--spacing`).
- Packing: shelf pack, widest-first, left‚Üíright, wrap to a new shelf, then a new
  plate if it overflows. Simple and deterministic.

Re-run for a smaller printer, e.g. a 220 mm Ender bed:
```bash
python make_print_plates.py --bed 220 220
```
At 220√ó220 everything still lands on the same 3 plates (rigid footprint is only
~240√ó112 ‚ÄĒ it re-packs onto a couple more shelves but fits).

## Part quantities (full assembly = 17 printed parts)

| Part | Qty | Why |
|---|---|---|
| enclosure | 1 | one housing |
| front_cover | 1 | one snap-on cover |

| drive_arm_R | 1 | right arm; rides on `snap_pin_axle` |
| follower | 2 | one per side (L/R link bars are congruent ‚Üí same file) |
| snap_pin_axle | **4** | the 4 internal axle pivots (`pin_A_R`, `pin_A_L`, `pin_B_R`, `pin_B_L`) |
| snap_pin_finger | **4** | the 4 finger-pivot pins (`pin_C_R`, `pin_D_R`, `pin_C_L`, `pin_D_L`) |
| finger_R | 1 | right Fin Ray finger (chiral) |
| finger_L | 1 | left Fin Ray finger (chiral mirror) |
| input_pinion_shaft | 1 | pinion + vertical shaft + capture collar + D-coupler (new right-angle drive) |
| **Total** | **17** | **3 plates** (1 rigid PA12-GF + 1 PETG-HF + 1 TPU) |

`pin_A_L` is now a separate snap-pin axle (same geometry as `pin_A_R`) ‚ÄĒ the
left arm no longer carries an integral horizontal shaft. The right-angle drive
input shaft is `input_pinion_shaft`, a separate printed part. Zero bought hardware.

- **Solid material volume: ‚Čą 103.1 cm¬≥** (geometric solid volume, summed over all
  17 parts, BEFORE walls/infill ‚ÄĒ actual filament will be lower for the

- **Print-time estimate: not produced ‚ÄĒ no slicer CLI was found on PATH.** The

  time + filament grams (it slices each plate at `--layer`, default 0.2 mm).
